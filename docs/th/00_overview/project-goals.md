# เป้าหมายโปรเจกต์

## วิสัยทัศน์
เพื่อสร้างสุดยอด desktop-first IDE สำหรับการพัฒนา AI โดยมอบอำนาจให้ผู้พัฒนาสามารถสร้าง, ทดสอบ และจัดการ Claude AI skills และ agents ที่ซับซ้อนด้วยความปลอดภัยระดับองค์กรและการออกแบบตามหลักสรีรศาสตร์สำหรับนักพัฒนา

## วัตถุประสงค์หลัก (Core Objectives)

### 1. Unified AI Development Interface
จัดเตรียมสภาพแวดล้อมเดียวที่ทรงพลังเพื่อแทนที่เครื่องมือที่กระจัดกระจายสำหรับการทำ prompt engineering, การจัดการ skill และการทดสอบโมเดล

### 2. ความปลอดภัยระดับองค์กร (Enterprise-Grade Security)
รับประกันว่า API credentials และข้อมูลที่ละเอียดอ่อนทั้งหมดจะถูกจัดเก็บในเครื่อง (locally) และเข้ารหัสโดยใช้มาตรฐานอุตสาหกรรม AES-256-GCM เพื่อให้ผู้ใช้สามารถควบคุมความลับของตนได้อย่างเต็มที่

### 3. ความยืดหยุ่นในการรองรับหลาย Provider (Multi-Provider Flexibility)
รองรับผู้ให้บริการ LLM ที่หลากหลาย เช่น AWS Bedrock, Anthropic, OpenAI, Google, OpenRouter และ local models (Ollama, LM Studio) เพื่อให้แน่ใจว่านักพัฒนาจะไม่ถูกผูกติดอยู่กับระบบนิเวศเพียงระบบเดียว

### 4. ประสิทธิภาพของนักพัฒนา (Developer Productivity)
มีคลัง agent ที่สร้างโดยผู้เชี่ยวชาญกว่า 501+ รายการ และ 15+ skill templates เพื่อเร่งการเริ่มต้นของโปรเจกต์ AI

### 5. การรับประกันคุณภาพที่แข็งแกร่ง (Robust Quality Assurance)
จัดเตรียมเครื่องมือการทดสอบที่รวมอยู่ในตัว รวมถึงการตรวจสอบความถูกต้องอัตโนมัติสำหรับ skills และ agents และชุด E2E tests กว่า 80+ รายการเพื่อให้มั่นใจในความน่าเชื่อถือของแพลตฟอร์ม

## หลักการ (Principles)

- **Human-in-the-loop**: การดำเนินการของ AI ทั้งหมดจะต้องผ่านการพิจารณาและอนุมัติโดยผู้ใช้
- **Privacy First**: สถาปัตยกรรมแบบ local-first ช่วยให้มั่นใจได้ว่าข้อมูลของผู้ใช้จะอยู่ในเครื่องของตนเว้นแต่จะมีการแชร์อย่างชัดเจน
- **Type-Safe Everything**: ใช้ประโยชน์จาก TypeScript 5.8 และ tRPC เพื่อความปลอดภัยของประเภทข้อมูลแบบ end-to-end ตลอดทั้งระบบ
- **Open Standards**: รองรับ Model Context Protocol (MCP) และข้อกำหนด OpenAPI 3.1
